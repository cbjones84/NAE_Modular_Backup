# AutoGen Studio Files Reference

This document explains the purpose of each AutoGen Studio related file to avoid duplicates.

## Primary Integration Files (Use These)

### `autogen_studio_nae_integration.py` ⭐ **PRIMARY**
- **Purpose**: Main integration for AutoGen Studio with NAE
- **Features**: 
  - NAEStudioIntegration class
  - All NAE agents (Casey, Optimus, Ralph, Donnie, Genny)
  - Workflow configuration generation
  - Agent bridge system
- **Use this for**: New integrations, workflow generation

### `autogen_studio_setup.py` ⭐ **PRIMARY**
- **Purpose**: Setup script for AutoGen Studio configuration
- **Features**: Creates agent configs, workflow configs, function definitions
- **Use this for**: Initial setup and configuration generation

## Supporting Files

### `autogen_studio_bridge.py`
- **Purpose**: Flask API bridge server for AutoGen Studio
- **Features**: REST API endpoints for Casey agent communication
- **Dependencies**: Uses `autogen_studio_nae_integration.py` (preferred) or `autogen_studio_integration.py` (fallback)
- **Use this for**: Running API bridge server

### `scripts/start_autogen_studio_nae.sh`
- **Purpose**: Startup script for AutoGen Studio with NAE
- **Use this for**: Quick startup

## Legacy Files (Kept for Compatibility)

### `autogen_studio_integration.py` (Legacy)
- **Purpose**: Original integration (Casey-focused)
- **Status**: Kept for backward compatibility with `autogen_studio_bridge.py`
- **Note**: New code should use `autogen_studio_nae_integration.py`

### `setup_autogen_studio.py` (Legacy)
- **Purpose**: Original setup script
- **Status**: Superseded by `autogen_studio_setup.py`
- **Note**: Use `autogen_studio_setup.py` instead

## Configuration Files

### `.autogenstudio/agents.json`
- **Purpose**: Generated agent configurations
- **Status**: Auto-generated, in `.gitignore`
- **Generated by**: `autogen_studio_nae_integration.py`

### `.autogenstudio/workflows.json`
- **Purpose**: Generated workflow configurations
- **Status**: Auto-generated, in `.gitignore`
- **Generated by**: `autogen_studio_nae_integration.py`

### `autogen_studio_config.json`
- **Purpose**: Manual configuration file
- **Status**: Can be used for custom configurations

### `autogen_functions.json`
- **Purpose**: Function definitions for AutoGen Studio
- **Status**: Used by setup scripts

### `autogen_workflow.json`
- **Purpose**: Workflow configuration template
- **Status**: Template file

## Best Practices

1. **For new integrations**: Use `autogen_studio_nae_integration.py`
2. **For setup**: Use `autogen_studio_setup.py`
3. **For API bridge**: Use `autogen_studio_bridge.py` (it will use the best available integration)
4. **For documentation**: Refer to `AUTOGEN_STUDIO_NAE_INTEGRATION.md` and `QUICK_START_AUTOGEN_STUDIO.md`

## Avoiding Duplicates

- ✅ Keep both integration files (old and new) for compatibility
- ✅ Use new files for new development
- ✅ Generated configs (`.autogenstudio/`) are in `.gitignore`
- ✅ Legacy files are kept for backward compatibility
- ❌ Don't create new files that duplicate existing functionality
- ❌ Don't commit generated config files
