# Agent Definitions

This directory contains JSON or YAML definitions for dynamically loaded NAE agents. 
These files are typically generated by **Flowise** or created manually for testing.

## Schema

Each agent definition should follow this structure:

```json
{
  "name": "AgentName",
  "version": "1.0.0",
  "enabled": true,
  "class": "agents.base_dynamic_agent.DynamicAgent",
  "description": "Brief description of what this agent does",
  "inputs": ["market_data_feed", "risk_signals"],
  "outputs": ["trade_execution", "strategy_signals"],
  "config": {
    "strategy_type": "mean_reversion",
    "risk_level": "low",
    "timeframe": "1h",
    "symbols": ["AAPL", "TSLA", "BTC-USD"],
    "parameters": {
       "lookback_period": 20,
       "entry_threshold": 2.0,
       "exit_threshold": 0.5
    }
  }
}
```

## Fields

- **name**: Unique identifier for the agent (must be unique across system).
- **version**: Semantic versioning string. NAE can handle upgrades/rollbacks based on this.
- **enabled**: Boolean to quickly toggle the agent.
- **class**: (Optional) Fully qualified path to the Python class to instantiate. Defaults to `agents.base_dynamic_agent.DynamicAgent`.
- **description**: Human-readable description.
- **inputs**: List of data streams or message types this agent subscribes to.
- **outputs**: List of message types this agent publishes.
- **config**: Dictionary of configuration parameters passed to the agent's `__init__` or `configure` method.

## Hot-Swapping

The `AgentLoader` watches this directory.
- **Adding** a new file will instantiate and register the agent.
- **Modifying** an existing file will trigger a reload (stop -> re-instantiate -> start).
- **Deleting** a file will stop and unregister the agent.
